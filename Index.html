Overview
Purpose
Tracks all IT development activities for Madaen Star Group. Four tabs: Dashboard, Activity Log, Weekly Update, User Guide.
Who updates
Mubashir Hasan — updates Activity Log daily. Completes and submits Weekly Update every Thursday.
Who reviews
Management reviews the Dashboard and Weekly Update tabs.
Data storage
All data is saved automatically in your browser. Do not clear browser data or use incognito mode — this will erase your task history.
Tab 1 — Dashboard
Purpose
Executive summary view. All data is automatic — do not edit directly.
KPI Cards
Six cards update live: Total, Completed, In Progress, High Priority Open, Overdue, and Missing Close Date.
Monthly Chart
Shows Added, Completed, and Pending per month (non-cumulative). Added = Date Added field. Completed = Actual Close Date field. Pending = tasks In Process at end of that month.
Flag Sections
High Priority (open tasks), Overdue (past target), and Missing Close Date (done tasks without close date) — all update automatically.
Tab 2 — Activity Log
Adding a task
Fill in the form at the top and click "Add Task". The task appears immediately at the top of the table. No row insertion needed.
Reference number
Follow the sequence: 2026/016, 2026/017 etc. The next reference number is suggested automatically.
Date format
Use the date picker — dates are stored and displayed as DD-MMM-YYYY (e.g. 01-Apr-2026).
Days Open
Calculated automatically. Done + close date = close date minus start date. Open tasks = today minus start date. Done without close date = shows ⚠.
Actual Close Date
MANDATORY when marking a task Done. Without it the monthly chart cannot count completions and the dashboard will show it as missing.
Editing a task
Click the Edit button on any row to load it back into the form. Make changes and click Update Task.
Tab 3 — Weekly Update
When to complete
Every Thursday before submitting to management.
Week-end date
Select the Thursday date using the date picker. Sections 1, 2, and 5 update automatically.
Auto sections
Sections 1 (Summary counts), 2 (Done tasks list), and 5 (High Priority open tasks) populate from the Activity Log automatically.
Manual sections
Sections 3 (Blockers), 4 (Next Week Plan), and narrative fields must be typed manually each week.
Export
Click "Export as HTML" to save a standalone printable report. Click "Print Report" to print directly.
Colour Coding Legend
● Green — Done / Completed ● Amber — In Process / normal ● Red — High Priority or Overdue ● Orange — Done, missing close date ● Gray — Low priority ● Blue — On Hold
Important Rules
Close date is mandatory
Always fill Actual Close Date when marking a task Done. The chart and dashboard depend on this field.
Do not clear browser data
All tasks are stored in browser localStorage. Clearing browser data will permanently delete all tasks.
Use the same browser
Data is stored per browser. Always open the file in the same browser to see your data.
Backup regularly
Use "Export Report" from the top bar to save a backup of your data periodically.
